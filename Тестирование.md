# План тестирования АналогJira

## 1. Виды тестирования

| Вид | Инструмент | Что проверяем |
|-----|------------|---------------|
| Модульное (Unit) | Jest | Функции регистрации, создания задач, расчёта спринтов |
| Интеграционное | Postman | API: создание задач, WebSocket-уведомления |
| UI (пользовательское) | Selenium | Drag-and-drop, кнопки, формы, диаграммы |
| Нагрузочное | k6 | Работа при 100+ пользователях одновременно |

## 2. Тест-кейсы

| ID | Тест-кейс | Ожидаемый результат |
|----|-----------|---------------------|
| TC1 | Пользователь вводит email и пароль → нажимает «Войти» | Открывается главная страница |
| TC2 | Пользователь создаёт новую задачу | Задача появляется в списке |
| TC3 | Пользователь перетаскивает задачу в другую колонку | Задача меняет статус |
| TC4 | Пользователь добавляет комментарий к задаче | Комментарий отображается |
| TC5 | Пользователь завершает спринт | Строится диаграмма сгорания задач |

## 3. Покрытие тестами

- Модульные тесты: цель 80% кода
- Критические функции (авторизация, создание задач): 100%
