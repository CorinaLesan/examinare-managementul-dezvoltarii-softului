# SRS: АналогJira — приложение для управления проектами

## 1. Введение

### 1.1 Цель документа
Описать функциональные требования в формате User Stories и нефункциональные требования к веб-приложению AnalogJira.

### 1.2 Целевая аудитория
- Команды разработчиков, использующие Agile/Scrum
- Менеджеры проектов
- Администраторы системы

## 2. User Stories (функциональные требования)

| ID | User Story |
|----|------------|
| US1 | Как пользователь, я хочу зарегистрироваться и войти в систему, чтобы иметь доступ к проектам |
| US2 | Как пользователь, я хочу создавать, редактировать и удалять задачи в проекте |
| US3 | Как пользователь, я хочу перетаскивать задачи по колонкам Kanban-доски (Drag-and-Drop) |
| US4 | Как пользователь, я хочу настраивать статусы колонок (To Do, In Progress, Done) |
| US5 | Как пользователь, я хочу создавать спринты и планировать задачи из бэклога |
| US6 | Как пользователь, я хочу видеть диаграмму сгорания задач (Burndown Chart) |
| US7 | Как пользователь, я хочу оставлять комментарии и прикреплять файлы к задачам |
| US8 | Как пользователь, я хочу получать уведомления в реальном времени (WebSocket) |
| US9 | Как администратор, я хочу видеть отчёты по скорости команды (Velocity) |
| US10 | Как пользователь, я хочу управлять зависимостями между задачами |

## 3. Нефункциональные требования (NFR)

| ID | Требование |
|----|-------------|
| NFR1 | Время отклика интерфейса не более 2 секунд |
| NFR2 | Пароли пользователей хранятся в зашифрованном виде (bcrypt) |
| NFR3 | Приложение должно корректно работать в Chrome, Firefox, Edge |
| NFR4 | Данные пользователя доступны только ему |
| NFR5 | WebSocket-соединение должно восстанавливаться при обрыве |

## 4. Ограничения
- Стек технологий: React, Node.js, MongoDB, WebSocket
- Методология: Agile/Scrum
- Разработка ведётся индивидуально
